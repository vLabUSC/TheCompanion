Creates a new instance of a Blueprint class and places it in the world at runtime. This is how you make things appear during gameplay — projectiles, enemies, pickups, effects, anything that doesn't exist in the level at start.

## Use When

- You need to create actors during gameplay (fire a projectile, spawn an enemy wave, drop a pickup)
- The actor shouldn't exist in the level beforehand — it's generated by game logic
- You need to place something at a specific location and rotation determined at runtime
- You want to create multiple copies of the same Blueprint class

## How It Works

### Input Pins

| Pin | Type | Description |
|---|---|---|
| **Class** | Class | The Blueprint class to spawn. Select from the dropdown or promote to a [[Variables|variable]] for runtime selection |
| **Spawn Transform** | Transform | Location, rotation, and scale for the new actor. Wire in a [[Get Actor Location]] or Make Transform node |
| **Collision Handling Override** | Enum | What to do if the spawn location is blocked: Always Spawn (ignore collisions), Try To Adjust (nudge to fit), or Do Not Spawn |
| **Instigator** | Actor | Optional — the actor responsible for spawning (used for damage attribution, etc.) |
| **Owner** | Actor | Optional — the actor that owns the spawned actor. The spawned actor can call Get Owner to get back to its creator |

### Output Pins

| Pin | Type | Description |
|---|---|---|
| **Return Value** | Actor | A reference to the newly created actor. Cast this to the specific class to access its variables and functions |

### Expose on Spawn

If the spawned Blueprint has [[Variables]] marked **Expose on Spawn**, those variables appear as extra input pins on the Spawn Actor node. This lets you configure the new actor at creation time — set its speed, color, target, or any other property before it begins playing.

## Common Patterns

**Projectile spawning:**
On an input event (mouse click, key press), call Spawn Actor from Class with the projectile Blueprint. Set Spawn Transform to the weapon muzzle location and forward rotation. The projectile Blueprint handles its own movement and destruction on hit via [[OnComponentHit]].

**Enemy wave:**
A spawner Blueprint uses a [[Flow Control|ForLoop]] or timer to call Spawn Actor multiple times. Each call creates an enemy at a slightly different location. Store spawned references in an array [[Variables|variable]] to track them (count remaining, destroy all on wave end).

**Configured spawning with Expose on Spawn:**
A pickup Blueprint has an Expose on Spawn variable `PointValue` (Integer). The spawner calls Spawn Actor and sets `PointValue` to 10, 25, or 100 per instance. Each spawned pickup knows its own value without needing a separate Blueprint per point tier.

**Getting back to the spawner:**
Pass `Self` to the **Owner** pin. The spawned actor can call **Get Owner** → [[Cast To]] the spawner class to access the spawner's variables or call its [[Custom Events]]. Useful for projectiles reporting hits back to the weapon that fired them.
